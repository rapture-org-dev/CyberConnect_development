CyberConnect — Future Features Roadmap & Ticket Priority Guide
================================================================================
Version: 2026-05-31
Audience: You create Tasks-sheet tickets yourself from this doc, then implement
          one ticket at a time (always ticket-driven).

Handbook habits (still apply when YOU write the ticket):
  - Ticket body = WHY + acceptance criteria + must-not cases + golden examples.
  - One ticket = one shippable slice (not a whole epic).

  - Billing/accounting are large and risky; do after planning views exist.

C1. Change color of expired task row          [Priority 1]
------------------------------------------------------------------------------
Status: NOT IMPLEMENTED
Complexity: S (1–2 days)
Depends on: none

Why:
  PM/devs must see overdue work at a glance on the Tasks sheet.

Who / when:
  Anyone with Tasks tab access; JA/EN UI; compares deadline vs today (local TZ).

Definition of done:
  - Row background or left border distinct when deadline < today AND status not Done
  - Done / empty deadline: no overdue styling
  - Styling works in sticky header + scroll grid (GenericSheet)
  - Optional: legend in toolbar ("Overdue")

Red team:
  - Must not mark completed tasks overdue
  - Must not break row selection / batch delete / inline edit colors

Data model:
  None (uses task_rows.deadline, task_rows.status)

Implementation steps:
  1. Add helper: isTaskRowOverdue(row, today) in src/lib/ or next to tasks utils
  2. In GenericSheet <tr>, when tab.id === 'tasks', append overdue className
  3. Tailwind: e.g. bg-red-950/30 or border-l-2 border-red-500/60
  4. Document rule in code comment (WHY: business rule for "expired")
  5. Golden tests: 5 rows (past deadline + In progress, past + Done, future, blank)

Files to read:
  src/components/GenericSheet.tsx, src/lib/data.ts (isTasksTab), task status enum

C3. Sprint implementation function (structured sprints)  [Priority 2]
------------------------------------------------------------------------------
Status: PARTIAL (sprint is free text on tasks/backlog)
Complexity: L (1–2 weeks)
Depends on: none (blocks burndown, kanban grouping, better Gantt)

Why:
  Real sprint planning: start/end, name, goal, velocity — not only a string column.

Who / when:
  PM creates sprints per project; devs assign tasks to a sprint.

Definition of done:
  - Table project_sprints (id, project_id, name, name_ja, start_date, end_date, goal, status)
  - CRUD API under /api/projects/[id]/sprints
  - Tasks: sprint_id FK OR validated dropdown synced with sprint names
  - UI: "Sprints" sub-page or modal from Tasks toolbar
  - Migrating existing sprint text → best-effort match on first save

Red team:
  - RLS: only project members see sprints
  - Deleting sprint must not delete tasks (set null or block)

Data model:
  project_sprints + task_rows.sprint_id (migration from text sprint)

Implementation steps:
  1. SQL migration + RLS policies (mirror project_members pattern)
  2. server/sprints.ts + route handlers
  3. Replace/enhance sprint column type 'select' with dynamic options from API
  4. WorkspaceProvider cache invalidation on sprint change
  5. Golden: create 2 sprints, assign tasks, filter tasks by sprint

Files to read:
  database/cyberconnect_schema.sql (task_rows), src/server/rows.ts, data.ts tasks tab

================================================================================
PART G — JIRA PARITY GAP (QA VIEW)
================================================================================

Purpose:
  Assume Part A priorities 1–11 (roadmap features) are FULLY implemented.
  This section lists what is STILL missing to be "industry standard like Jira"
  — and what CyberConnect is already stronger at.

Two different "done" bars (use in reviews):
  [Delivery tool]  JP/EN PM runs requirements + tasks + tests + schedule in one
                   project better than Jira + Excel → roadmap + polish ≈ close.
  [Jira replacement] 50-person org drops Jira entirely → gaps below remain;
                   treat as a multi-wave program AFTER Part A.

--------------------------------------------------------------------------------
G0. AFTER ROADMAP — WHAT YOU WOULD HAVE (SUMMARY)
--------------------------------------------------------------------------------

  Strong:
    - Task grid + Kanban + overdue styling + sprints + burndown + Gantt slice
    - Bilingual sheets (purpose, screens, functions, tasks, tests, APIs, backlog)
    - Import/export, column layouts, resize, DeepL, team/project RBAC + RLS
    - Billing/leads/accounting starters (not Jira core)

  Weak vs Jira:
    - No universal "issue" product with workflow engine and marketplace
    - No cross-project JQL, dashboards, automation rules, notification fabric
    - No enterprise SSO/audit/compliance product story at Atlassian scale

--------------------------------------------------------------------------------
G1. ISSUE MODEL & HIERARCHY (HIGH GAP)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Issue types (Story, Bug, Sub-task) with different fields/screens per type
    - Parent/child hierarchy + link types (blocks, duplicates, relates, epic link)
    - Sub-tasks with rolled-up remaining work
    - Move/clone issue across projects with field mapping
    - Cross-project issue search by key

  CyberConnect today:
    - Sheet rows + codes (task_code, screen_code, etc.), epic as columns not a graph

  Future wave (if pursuing parity): "Work item hub" — work_items table + links table

--------------------------------------------------------------------------------
G2. WORKFLOW ENGINE (HIGH GAP)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Custom workflows: states, transitions, conditions, validators
    - Post-functions (auto-assign, webhooks on transition)
    - Per-project / per-issue-type workflow schemes

  CyberConnect today:
    - Fixed status enums on sheet rows; no admin-configurable state machine

  Future wave: workflow_definitions + transition_permissions (admin UI)

--------------------------------------------------------------------------------
G3. PERMISSIONS & GOVERNANCE (HIGH FOR ENTERPRISE)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Permission schemes (browse/create/edit/transition/delete/administer)
    - Issue-level and field-level security
    - Admin audit log (who changed workflow, scheme, project config)
    - SSO/SAML/OIDC, SCIM, org policies, compliance packaging (SOC2, DPA, residency)

  CyberConnect today:
    - team/project RBAC, RLS on rows — good for single-tenant team model, not Jira schemes

  Future wave: permission_scheme per project; config_audit_log table

--------------------------------------------------------------------------------
G4. SEARCH, FILTERS, DASHBOARDS (HIGH GAP)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - JQL-like query across projects and fields
    - Saved filters, subscriptions, shared filters
    - Personal dashboards + gadgets (assigned to me, sprint health, etc.)
    - @mentions, watchers, rich comment thread per issue
    - Attachments with preview on the work item

  CyberConnect today:
    - One tab grid per sheet; sort/filter in UI only, no global issue index

  Future wave: saved_filters + global_search API; comments/attachments on work_items

--------------------------------------------------------------------------------
G5. AGILE MECHANICS BEYOND ROADMAP (MEDIUM–HIGH GAP)
--------------------------------------------------------------------------------

  Missing vs Jira (even after burndown + Kanban + sprints):
    - Backlog rank (drag order independent of sheet row order)
    - Sprint planning: capacity, commitment, complete sprint, spillover handling
    - Kanban WIP limits, swimlanes (assignee, epic, priority)
    - Reports: velocity, control chart, cumulative flow, sprint report, release burndown
    - Story points as first-class metric (vs person_day only)
    - Fix version / affects version tied to releases

  Future wave: backlog_rank column; report/* endpoints; WIP on board config

--------------------------------------------------------------------------------
G6. PORTFOLIO & PROGRAM MANAGEMENT (HIGH FOR MULTI-TEAM)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Cross-project roadmaps and epic timelines
    - Dependency graph and critical path across teams
    - Capacity allocation across people/teams
    - Scenario / what-if planning

  CyberConnect today:
    - Per-project schedule tab; master_schedule placeholder — not Advanced Roadmaps

  Future wave: program entity linking multiple projects; dependency_links table

--------------------------------------------------------------------------------
G7. NOTIFICATIONS & COLLABORATION (HIGH GAP)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Email + in-app notifications (assigned, mentioned, status, due soon)
    - Per-user notification preferences
    - Slack / Microsoft Teams integrations
    - Deep link issue ↔ doc (Confluence-style); activity feed per item

  Future wave: notification_events + user_notification_settings + email provider

--------------------------------------------------------------------------------
G8. AUTOMATION & INTEGRATIONS (VERY HIGH GAP)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Rule builder (when field X → action Y)
    - Full admin REST surface for all config objects
    - Outbound webhooks on every relevant event (not only inbound sheet sync)
    - Marketplace / app ecosystem (GitHub PR, test tools, etc.)
    - CI/CD: deployments visible on work items, release boards

  CyberConnect today:
    - Inbound sheet sync (planned C7); GitHub/Slack not productized

  Future wave: automation_rules table; outbound_webhooks; github_integration MVP

--------------------------------------------------------------------------------
G9. TIME TRACKING & ESTIMATION (MEDIUM GAP)
--------------------------------------------------------------------------------

  Missing vs Jira:
    - Worklogs per issue, timesheets, approval
    - Original estimate / remaining / spent roll-ups
    - Billable hours from worklogs (different from team Stripe billing)

  CyberConnect today:
    - person_day on task rows — useful but not Tempo-class tracking

  Future wave: work_logs table linked to task_rows or work_items

--------------------------------------------------------------------------------
G10. NON-FUNCTIONAL / QA BAR (ENTERPRISE PROCUREMENT)
--------------------------------------------------------------------------------

  | Area              | Jira Cloud expectation     | Gap if claiming "like Jira"      |
  |-------------------|----------------------------|----------------------------------|
  | Scale             | Huge issue count, fast JQL | Indexing, load tests, pagination |
